
redis数据类型
	Redis中的key一定是字符串，value可以是string、list、hash、set、sortset
	字符串 、散列、列表、集合、有序集合
	String、Hash、List、   Set、    zSet
应用场景
	String 存储单个值，如论坛网站记录登录用户数（一个String类型的value最大可以存储512M）
	Hash  键值对集合，适合存储化对象（如加油站对象stationInfo）
	List   论坛网站记录登录用户；当队列使用（先进先出）
	set   没有重复的集合，可以做全局去重
	zset 可以做排行榜（如小说网站）
zset数据结构
	压缩列表（有序的，zset数据量小于128并且大小小于64字节时使用）
	跳跃表（有序列表，每个节点存储元素和分值）
	字典（键值对，key为元素，value为分值。可以快速元素分值）

字典
	底层实现是hash表。字典有两个哈希表，一个在扩容时使用
